# Chinese Knot Guide 发文标准

> 适用范围：中国结站所有 tutorial、meaning、knot type、cord、bracelet、keychain、FAQ 和后续新增文章页。

## 1. 发文前必须具备的信息

| 字段 | 标准 |
|---|---|
| Primary Keyword | 一个主关键词，不与已发布页面冲突 |
| Search Intent | tutorial-diy / meaning-symbolism / knot-types / product-craft / general |
| URL Slug | 英文小写短横线，不堆词 |
| SEO Title | 35-78 字符，核心词靠前 |
| Meta Description | 90-165 字符，说明用户能得到什么 |
| Article Type | tutorial / meaning / type / supply / craft / FAQ |
| Related Internal Links | 3-5 个候选内链 |
| FAQ Plan | 2-5 个当前页面专属 FAQ |

## 2. 写作原则

- 先给直接答案，再补背景。
- tutorial 页面必须讲清材料、难度、步骤顺序、常见错误。
- meaning 页面必须说明文化寓意，不能写成确定的命运、运势或保证效果。
- product-craft 页面只做选购、材料、场景、礼物用途说明，前期不硬推商品。
- 避免 AI 套话：Firstly、Secondly、Finally、It is worth noting that、In conclusion、This article explores。
- 段落短，表格清晰，FAQ 直接回答问题。

## 3. 正文结构

推荐结构：

```text
直接答案
-> 使用场景 / 材料 / 难度
-> 核心解释或步骤顺序
-> 对比表格
-> 常见错误
-> Related Guides
-> FAQ
-> 结尾引导到下一个工具或指南
```

## 4. SEO/GEO 标准

| 项目 | 标准 |
|---|---|
| H1 | 每页仅 1 个 |
| H2 | 至少 2 个 |
| FAQ | 详情页 2-5 个专属 FAQ |
| Canonical | 必须指向当前正式 URL |
| Sitemap | 发布后进入 sitemap |
| 内链 | 每篇 3-5 条 |
| 短答案 | 页面内至少 1 段可直接引用 |

## 5. 合规边界

- 不承诺招财、转运、保平安等确定结果。
- 不使用侵权图片、品牌图、平台商品图。
- 不把佛教、道教、民俗符号混写成确定宗教结论。
- 不写 guaranteed luck、this knot will bring wealth 等绝对化表达。
